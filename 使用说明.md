# RustDesk 服务器用户和设备管理功能使用说明

## 📋 概述

本项目为 RustDesk 服务器添加了完整的用户和设备管理功能，支持用户注册、设备分配、权限管理等企业级功能。

## 🚀 快速开始

### 1. 编译项目

```bash
# 清理之前的构建
cargo clean

# 编译发布版本
cargo build --release

# 编译成功后，可执行文件位于：
# - target/release/hbbs (主服务器)
# - target/release/hbbr (中继服务器)
# - target/release/rustdesk-utils (工具集)
```

### 2. 环境配置

创建 `.env` 文件或设置环境变量：

```bash
# JWT 密钥（生产环境必须设置）
export JWT_SECRET="your-super-secret-jwt-key-change-in-production"

# 数据库文件路径（可选，默认为 ./db_v2.sqlite3）
export DB_URL="./data/rustdesk_server.db"

# 数据库连接池大小（可选，默认为 1）
export MAX_DATABASE_CONNECTIONS="5"
```

### 3. 启动服务

```bash
# 方式一：直接启动
./target/release/hbbs

# 方式二：使用环境变量启动
JWT_SECRET="your-secret-key" ./target/release/hbbs

# 方式三：使用配置文件启动
./target/release/hbbs -c config.toml
```

服务启动后会显示：
```
INFO  数据库初始化完成: ./db_v2.sqlite3
INFO  API 服务器启动在端口: 8080
INFO  hbbs start
```

## 🌐 Web 管理界面

### 访问地址
- **管理界面**: http://localhost:8080
- **API 文档**: http://localhost:8080/api

### 首次使用

1. **创建管理员账户**
```bash
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "email": "admin@example.com",
    "password": "admin123456"
  }'
```

2. **登录管理界面**
- 打开浏览器访问 http://localhost:8080
- 使用刚创建的管理员账户登录

### 功能模块

#### 📊 仪表板
- 总用户数统计
- 总设备数统计
- 活跃设备数统计

#### 👥 用户管理
- **查看用户列表**: 显示所有注册用户
- **创建用户**: 添加新的用户账户
- **编辑用户**: 修改用户信息（用户名、邮箱）
- **删除用户**: 删除用户及其所有设备

#### 🖥️ 设备管理
- **设备列表**: 查看所有已分配的设备
- **添加设备**: 将 RustDesk 设备分配给用户
- **移除设备**: 从用户账户移除设备
- **设备搜索**: 按设备ID或用户名搜索
- **设备限制**: 每个用户最多10个设备

## 🔌 API 接口文档

### 认证接口

#### 用户登录
```bash
POST /api/login
Content-Type: application/json

{
  "username": "用户名",
  "password": "密码"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "token": "jwt-token-here",
    "user": {
      "id": 1,
      "username": "admin",
      "email": "admin@example.com",
      "created_at": "2024-01-01T00:00:00Z",
      "is_active": true
    }
  },
  "message": "操作成功"
}
```

### 用户管理接口

#### 获取用户列表
```bash
GET /api/users?limit=10&offset=0
Authorization: Bearer {jwt-token}
```

#### 创建用户
```bash
POST /api/users
Authorization: Bearer {jwt-token}
Content-Type: application/json

{
  "username": "newuser",
  "email": "user@example.com",
  "password": "password123"
}
```

#### 获取用户详情
```bash
GET /api/users/{user_id}
Authorization: Bearer {jwt-token}
```

#### 更新用户
```bash
PUT /api/users/{user_id}
Authorization: Bearer {jwt-token}
Content-Type: application/json

{
  "username": "updateduser",
  "email": "updated@example.com"
}
```

#### 删除用户
```bash
DELETE /api/users/{user_id}
Authorization: Bearer {jwt-token}
```

### 设备管理接口

#### 获取用户设备
```bash
GET /api/users/{user_id}/devices
Authorization: Bearer {jwt-token}
```

#### 添加设备
```bash
POST /api/users/{user_id}/devices
Authorization: Bearer {jwt-token}
Content-Type: application/json

{
  "user_id": 1,
  "device_id": "1234567890",
  "device_name": "办公电脑"
}
```

#### 移除设备
```bash
DELETE /api/users/{user_id}/devices/{device_id}
Authorization: Bearer {jwt-token}
```

#### 查询设备所有者
```bash
GET /api/devices/{device_id}/owner
Authorization: Bearer {jwt-token}
```

## 🔧 配置选项

### 服务器配置

在启动时可以使用以下参数：

```bash
./target/release/hbbs --help

# 常用参数：
-p, --port=[NUMBER]           # 设置 Rendezvous 服务器端口（默认 21115）
-k, --key=[KEY]             # 设置连接密钥
-R, --rendezvous-servers=[HOSTS] # 设置中继服务器
-u, --software-url=[URL]      # 设置软件下载地址
```

### 环境变量

| 变量名 | 描述 | 默认值 | 必需 |
|---------|------|--------|------|
| `JWT_SECRET` | JWT 签名密钥 | 无 | 是（生产环境） |
| `DB_URL` | 数据库文件路径 | `./db_v2.sqlite3` | 否 |
| `MAX_DATABASE_CONNECTIONS` | 数据库连接池大小 | `1` | 否 |

## 🛡️ 安全配置

### 生产环境建议

1. **设置强 JWT 密钥**
```bash
# 生成随机密钥
openssl rand -hex 32
# 或使用
JWT_SECRET="$(openssl rand -hex 32)"
```

2. **配置防火墙**
```bash
# 开放必要端口
ufw allow 21115/tcp  # Rendezvous 端口
ufw allow 8080/tcp   # API 管理端口
```

3. **使用 HTTPS**
```bash
# 建议在反向代理中配置 HTTPS
# nginx 配置示例：
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
}
```

## 📝 数据库结构

### users 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 用户唯一ID |
| username | VARCHAR(50) UNIQUE | 用户名 |
| email | VARCHAR(100) UNIQUE | 邮箱地址 |
| password_hash | VARCHAR(255) | 密码哈希值 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| is_active | BOOLEAN | 账户状态 |

### user_devices 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 关系记录ID |
| user_id | INTEGER | 所属用户ID |
| device_id | VARCHAR(100) | RustDesk 设备ID |
| device_name | VARCHAR(100) | 设备名称（可选） |
| created_at | DATETIME | 添加时间 |
| is_active | BOOLEAN | 设备状态 |

## 🔍 故障排除

### 常见问题

#### 1. 编译失败
```bash
# 确保安装了必要的依赖
cargo update
cargo build --release

# 如果仍有问题，清理重试
cargo clean && cargo build --release
```

#### 2. 数据库连接失败
```bash
# 检查数据库文件权限
ls -la db_v2.sqlite3

# 检查数据库路径
export DB_URL="./full/path/to/database.db"
```

#### 3. API 无法访问
```bash
# 检查端口是否被占用
lsof -i :8080

# 检查防火墙设置
ufw status
```

#### 4. 用户登录失败
- 检查用户名和密码是否正确
- 确认用户账户处于活跃状态
- 检查 JWT 密钥配置

### 日志查看

```bash
# 查看实时日志
./target/release/hbbs 2>&1 | tee rustdesk.log

# 查看错误日志
tail -f rustdesk.log | grep ERROR
```
